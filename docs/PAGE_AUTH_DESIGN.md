# FinAgents OS — 页面权限与登录状态展示方案

基于 FinAgents Operating System 现有技术栈（Next.js 16 + Supabase Auth + App Router），规划登录前后的页面展示策略与技术实现方案。

> **系统全称**：FinAgents Operating System（FinAgents OS）
> **定位**：金融智能代理操作系统，提供 FAQ 知识库、AI 聊天、数据分析等功能模块
> **当前阶段**：先落地 FAQ 模块，后续逐步扩展
> **创建日期**：2026-03-03

---

## 1. 需求概述

| 场景       | 说明                                                              |
| ---------- | ----------------------------------------------------------------- |
| 未登录用户 | 可浏览公开 FAQ 知识库、使用基础聊天等公开功能                     |
| 登录后用户 | 在公开功能基础上，增加查询、分析等需要三方服务 API Key 的高级功能 |
| 布局策略   | 登录前后共享同一导航栏布局，不需要完全不同的页面结构              |
| 增强模式   | 登录后在现有页面上"叠加"更多功能入口，而非跳转到完全不同的页面    |
| 移动端优先 | 主要使用场景为移动端，设计以 Mobile-First 为原则                  |

```

├── page.tsx                      # 首页 → FAQ 列表（Tab: 首页）
├── globals.css
│
├── login/                        # 登录页（公开，隐藏底部 Tab）
│   ├── page.tsx
│   └── login-form.tsx
│
├── faqs/                         # FAQ 相关（公开 + 增强）
│   └── [id]/
│       └── page.tsx              # FAQ 详情页
│
├── chat/                         # AI 聊天功能（Tab: 聊天）
│   └── page.tsx
│
├── profile/                      # 个人中心（Tab: 我的）— 公开但内容按登录状态区分
│   └── page.tsx                  # 未登录：登录入口 / 已登录：个人菜单
│
├── (protected)/                  # Route Group：受保护路由
│   ├── analysis/                 # 查询/分析（Tab: 分析）
│   │   └── page.tsx
│   ├── my-faqs/                  # 我的 FAQ 管理
│   │   └── page.tsx
│   └── settings/                 # 个人设置
│       └── page.tsx
│
└── components/                   # 页面级共享组件
    ├── top-navbar.tsx            # 桌面端顶部导航（lg+）
    ├── bottom-tabs.tsx           # 移动端底部 Tab 栏（<lg）
    └── mobile-header.tsx         # 移动端顶部栏（<lg）
```

**关键说明：**

- `(protected)` 是 Next.js Route Group，不会体现在 URL 中，仅用于代码组织
- Route Group 内的页面共享父级 `layout.tsx`，无需单独 layout
- 未来新增功能模块（如行情、策略等）只需在对应位置添加目录

## 4. 导航设计（Mobile-First）

> **设计原则**：移动端是主要使用场景，采用底部 Tab 导航（参考同花顺/富途牛牛 App），桌面端自适应为顶部导航栏。

### 4.1 移动端 — 底部 Tab 栏

底部固定 Tab 栏，图标 + 文字，单手可达：

| Tab  | 图标            | 未登录                   | 登录后           |
| ---- | --------------- | ------------------------ | ---------------- |
| 首页 | `Home`          | 显示                     | 显示             |
| 聊天 | `MessageCircle` | 显示                     | 显示             |
| 分析 | `BarChart3`     | **显示但点击跳登录**     | 显示             |
| 我的 | `User`          | **显示（展示登录入口）** | 显示（个人中心） |

- 固定 4 个 Tab，未登录时"分析"和"我的"Tab 仍显示，点击后引导登录
- 当前选中 Tab 图标 + 文字变为主色 `#2962ff`，未选中为 `#848e9c`
- Tab 栏高度 `56px`（含安全区），背景 `#141821`，顶部 `1px solid #2a2e39`

### 4.2 移动端 — 顶部栏

轻量级顶部栏，仅展示当前页面标题和少量操作：

| 元素 | 说明                                      |
| ---- | ----------------------------------------- |
| 左侧 | 返回箭头（子页面时）或 Logo（Tab 首页时） |
| 中间 | 当前页面标题                              |
| 右侧 | 搜索图标 / 操作按钮（视页面而定）         |

### 4.3 桌面端 — 顶部导航栏

屏幕宽度 ≥ `lg`（1024px）时，底部 Tab 隐藏，切换为顶部水平导航栏：

| 导航元素          | 未登录   | 登录后               |
| ----------------- | -------- | -------------------- |
| Logo（FinAgents） | 显示     | 显示                 |
| FAQ 浏览          | 显示     | 显示                 |
| 聊天入口          | 显示     | 显示                 |
| 查询/分析         | **隐藏** | 显示                 |
| 我的 FAQ          | **隐藏** | 显示                 |
| 登录按钮          | 显示     | **隐藏**             |
| 头像 + 下拉菜单   | **隐藏** | 显示（含设置、登出） |

**实现方式：** `layout.tsx` 是 Server Component，在服务端获取 `user` 对象，传递给导航组件（含 `BottomTabs` + `TopNavbar`）做条件渲染。

## 5. 技术实现方案

### 5.1 路由保护 — proxy.ts

在现有 `proxy.ts` 中扩展受保护路由列表：

```typescript
const protectedRoutes = ['/analysis', '/my-faqs', '/settings'];
```

未登录用户访问受保护路由时，重定向到 `/login?next=/analysis`（携带来源路径参数，登录后跳回）。

### 5.2 Layout 条件渲染

`app/[locale]/layout.tsx` 中获取用户信息，渲染移动端底部 Tab + 桌面端顶部导航：

```tsx
import {createClient} from '~/lib/supabase/server';

export default async function RootLayout({children, params}) {
    const supabase = await createClient();
    const {
        data: {user},
    } = await supabase.auth.getUser();

    return (
        <html>
            <body>
                {/* 桌面端顶部导航，lg 以上显示 */}
                <TopNavbar user={user} className="hidden lg:flex" />

                {/* 移动端顶部栏，lg 以下显示 */}
                <MobileHeader className="flex lg:hidden" />

                {/* 主内容区，底部留出 Tab 栏空间 */}
                <main className="pb-14 lg:pb-0">{children}</main>

                {/* 移动端底部 Tab 栏，lg 以下显示 */}
                <BottomTabs user={user} className="flex lg:hidden" />
            </body>
        </html>
    );
}
```

### 5.3 混合页面的条件渲染

FAQ 列表 / 详情等混合页面在 Server Component 中获取 `user`，按是否存在来渲染额外操作按钮：

```tsx
export default async function FaqDetailPage({params}) {
    const supabase = await createClient();
    const {
        data: {user},
    } = await supabase.auth.getUser();

    return (
        <article>
            {/* 所有用户可见的内容 */}
            <FaqContent faqId={params.id} />

            {/* 登录后可见的增强功能 */}
            {user && <FaqActions faqId={params.id} userId={user.id} />}
        </article>
    );
}
```

### 5.4 API 路由保护（三方服务调用）

需要 API Key 的三方服务调用通过 Server Action 或 Route Handler 代理，不暴露 Key 给前端：

```typescript
// actions/analysis.ts
'use server';

import {createClient} from '~/lib/supabase/server';

export async function runAnalysis(faqId: string) {
    const supabase = await createClient();
    const {
        data: {user},
    } = await supabase.auth.getUser();
    if (!user) throw new Error('Unauthorized');

    const result = await fetch('https://third-party-api.com/analyze', {
        headers: {'Authorization': `Bearer ${process.env.THIRD_PARTY_API_KEY}`},
        body: JSON.stringify({faqId}),
    });

    return result.json();
}
```

### 5.5 登录回跳

在 `proxy.ts` 路由保护中，将当前路径附带到登录页 URL：

```typescript
if (isProtectedRoute && !user) {
    const locale = pathname.match(/^\/(zh|en)(?=\/|$)/)?.[1] ?? 'zh';
    const callbackUrl = encodeURIComponent(pathnameWithoutLocale);
    return NextResponse.redirect(new URL(`/${locale}/login?next=${callbackUrl}`, request.url));
}
```

登录成功后在 `/auth/callback` 中读取 `next` 参数并跳转回来。

## 6. 整体数据流

```
用户请求
  │
  ▼
proxy.ts
  ├── i18n 路由处理
  ├── Supabase Session 刷新
  └── 路由保护检查
        ├── 公开路由 → 放行
        └── 受保护路由 + 未登录 → 重定向 /login?next=xxx
  │
  ▼
Layout (Server Component)
  ├── 获取 user
  └── 渲染导航栏（按登录状态条件渲染菜单项）
  │
  ▼
Page (Server Component)
  ├── 获取 user（混合页面需要）
  └── 渲染内容 + 按登录状态条件渲染增强功能按钮
  │
  ▼
Server Action / Route Handler
  └── 校验 user 身份 → 调用三方服务（Key 不暴露给前端）
```

## 7. 方案选型总结

| 决策项   | 选择                             | 原因                                          |
| -------- | -------------------------------- | --------------------------------------------- |
| 布局方式 | 单一 Layout + 导航栏条件渲染     | 登录前后布局结构一致，不需要两套 layout       |
| 路由组织 | `(protected)` Route Group        | 不影响 URL 结构，便于代码组织和批量管理       |
| 路由保护 | proxy.ts 集中拦截                | 已有基础设施，扩展 `protectedRoutes` 数组即可 |
| 混合页面 | 页面级 Server Component 条件渲染 | FAQ 浏览页公开访问，登录后叠加操作按钮        |
| API 保护 | Server Action 内校验 user        | 三方 API Key 不暴露给前端，服务端代理调用     |
| 登录回跳 | `next` 查询参数                  | 保存来源路径，登录后自动跳回                  |
| 导航模式 | 移动端底部 Tab + 桌面端顶部导航  | 移动端为主要使用场景                          |

- **无闪烁**：`suppressHydrationWarning` + `next-themes` 的 script 注入

### 8.7 响应式策略（Mobile-First）

**核心原则：** 默认样式针对移动端，通过 `md:` / `lg:` 前缀向上增强。

| 断点 | 宽度     | 变化                                                              |
| ---- | -------- | ----------------------------------------------------------------- |
| 默认 | < 768px  | 移动端：底部 Tab、单列、全宽卡片、无限滚动、FAB 悬浮按钮          |
| `md` | ≥ 768px  | 平板：FAQ 卡片双列、输入框更宽                                    |
| `lg` | ≥ 1024px | 桌面：**底部 Tab → 顶部导航栏**、聊天侧边栏显示、分页替代无限滚动 |
| `xl` | ≥ 1280px | 大屏：内容区 max-width 限制、三列网格可选                         |

**导航切换逻辑：**

- `< lg`：底部 Tab 栏 + 轻量顶部栏
- `≥ lg`：顶部水平导航栏，底部 Tab 隐藏

**触摸优化：**

- 所有可点击元素最小触摸区域 `44x44px`（符合 Apple HIG）
- 底部 Tab 每个 Tab 的点击区域为 `flex-1` 等分整个宽度
- 列表项右侧带 `▸` 箭头指示可点击
- FAQ 列表支持下拉刷新 + 上拉加载更多

### 8.8 动效规范

| 元素           | 动效            | 参数                                      |
| -------------- | --------------- | ----------------------------------------- |
| 按钮 tap/hover | 背景色过渡      | `transition-colors duration-150`          |
| 按钮 tap       | 轻微缩放        | `active:scale-[0.98]`（移动端触觉反馈）   |

### 8.9 Tailwind CSS 主题变量

在 `globals.css` 中，通过 `:root` / `.dark` 定义双套色值，`@theme` 引用 CSS 变量使工具类自动跟随主题：

```css
@import 'tailwindcss';

/* 浅色主题（默认） */
:root {
    --app-bg-base: #f8f9fb;
    --app-bg-card: #ffffff;
    --app-bg-hover: #f2f3f7;
    --app-bg-alt: #f5f6f8;
    --app-border: #e2e5ea;
    --app-border-hover: #cdd1d8;
    --app-text-primary: #1a1d26;
    --app-text-secondary: #6b7280;
    --app-text-disabled: #a0a5b0;
    --app-accent: #2962ff;
    --app-accent-hover: #1b4fd9;
    --app-success: #0ecb81;
    --app-danger: #f6465d;
    --app-warning: #d97706;
}

/* 深色主题 */
.dark {
    --app-bg-base: #0b0e11;
    --app-bg-card: #141821;
    --app-bg-hover: #1c2030;
    --app-bg-alt: #0f1318;
    --app-border: #2a2e39;
    --app-border-hover: #3a3f4b;
    --app-text-primary: #e6e8ec;
    --app-text-secondary: #848e9c;
    --app-text-disabled: #474d57;
    --app-accent: #2962ff;
    --app-accent-hover: #2979ff;
    --app-success: #0ecb81;
    --app-danger: #f6465d;
    --app-warning: #f0b90b;
}

/* Tailwind @theme 引用 CSS 变量，工具类自动跟随主题 */
@theme {
    --font-inter: 'Inter', sans-serif;
    --font-lexend: 'Lexend', sans-serif;
    --font-mono: 'JetBrains Mono', monospace;

    --color-bg-base: var(--app-bg-base);
    --color-bg-card: var(--app-bg-card);
    --color-bg-hover: var(--app-bg-hover);
    --color-bg-alt: var(--app-bg-alt);
    --color-border: var(--app-border);
    --color-border-hover: var(--app-border-hover);

    --color-text-primary: var(--app-text-primary);
    --color-text-secondary: var(--app-text-secondary);
    --color-text-disabled: var(--app-text-disabled);

    --color-accent: var(--app-accent);
    --color-accent-hover: var(--app-accent-hover);
    --color-success: var(--app-success);
    --color-danger: var(--app-danger);
    --color-warning: var(--app-warning);
}
```

**使用方式**：所有组件只需使用 `bg-bg-base`、`text-text-primary`、`border-border` 等语义化工具类，无需写 `dark:` 前缀，主题切换时自动适配。
